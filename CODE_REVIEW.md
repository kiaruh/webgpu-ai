# Code Review Findings

## 1. Main Landing Page & UI
**Files:** `packages/client/src/app/page.tsx`, `layout.tsx`

- **Strengths:**
    - Excellent use of Metadata and JSON-LD for SEO.
    - Clean, modern styling with Tailwind CSS.
    - Good separation of layout and page content.
- **Recommendations:**
    - **Navigation:** Replace `<a>` tags with Next.js `<Link>` components in `page.tsx` (`/ads`, `/ai`) to enable client-side transitions and prefetching.
    - **Metadata:** Update the default `description` in `layout.tsx`. It currently says "Generated by create next app", which might fallback on pages that don't override it.

## 2. Client-Side AI Integration
**Files:** `packages/client/src/features/ai/FastVLMRunner.tsx`

- **Status:** Skeleton / Mocked.
- **Findings:**
    - The actual ONNX Runtime logic is commented out.
    - UI handles loading states well (`idle`, `loading`, `ready`, `error`).
- **Recommendations:**
    - **Implementation:** Uncomment the `onnxruntime-web` import and logic.
    - **Assets:** Ensure the `.onnx` model files are placed in `packages/client/public/models/` or configured to be fetched from a CDN.

## 3. Real-time Chat Feature
**Files:** `packages/client/src/features/chat/ChatWidget.tsx`, `packages/server/src/chat/`

- **Strengths:**
    - Functional WebSocket setup.
    - Clean UI for the Chat Widget with animations.
- **Critical Issues:**
    - **Hardcoded URL:** `packages/client/src/hooks/useSocket.ts` has `http://localhost:3001` hardcoded. This will break upon deployment.
        - *Fix:* Use an environment variable like `process.env.NEXT_PUBLIC_API_URL`.
    - **Directory Structure:** The backend has a redundant nested structure: `packages/server/src/chat/chat/`.
        - *Fix:* Move files up one level to `packages/server/src/chat/`.

## 4. Configuration & Setup
- **Strengths:**
    - Solid Monorepo structure.
    - Dependencies are up to date (Next.js 16, React 19, NestJS 11).
    - `tsconfig.json` paths are correctly configured for `@/*`.
- **Recommendations:**
    - Ensure `.env` files are added to `.gitignore` (already seems so, but double check before adding secrets).
